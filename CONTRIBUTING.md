# Contributing

Thanks for helping improve AiPowerHouse UI.

## Development workflow

- Fork the repo and create a feature branch.
- Keep changes scoped and focused.
- Add/adjust tests when behavior changes.
- Ensure all checks pass locally before opening a PR.

## Local setup

```bash
npm install
npm run format
npm run lint
npm run typecheck
npm run test
```

## Commit & PR expectations

- Prefer small PRs with a clear problem statement and solution.
- Include screenshots for UI changes.
- Call out follow-ups explicitly (tech debt, docs, tracking issues).

## Security

If you discover a vulnerability, do not open a public issue. See `SECURITY.md`.
